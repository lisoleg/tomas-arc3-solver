## 二、边界层理论(GPCT) → Oracle搜索策略

### 2.1 GPCT核心

NP难问题中约束耦合集中在少数高梯度变量上(计算边界层BL_c, 规模k)。只需枚举2^k种边界层赋值，外区(OR)可用经典逻辑(UP/BFS)线性求解。总复杂度从O(2^n)降解为O(2^k·poly(n))。

### 2.2 Oracle v17就是GPCT的实例

| GPCT步骤 | Oracle v17对应 | 验证 |
|----------|---------------|------|
| S0 预处理: 构建变量-约束图 | LevelInfo: 解析墙壁/目标/切换器/推方块 | ✅ |
| S1 多值编码: L_n→L_2 | 切换器类型(rot/shape/color)→离散枚举 | ✅ |
| S2 去冗筛选: 删除冗余子句 | _block_other_switchers: 非目标切换器加回walls | ✅ |
| S3 语境冻结: 识别BL_c | type_order排列组合 = 边界层变量 | ✅ |
| S4 模态降解: 枚举BL_c + 外区UP | 排列组合枚举 + BFS路径规划 | ✅ |
| S5 逆映射 | 动作序列执行 | ✅ |

**关键发现**: 我们修复Level 5 BFS穿越问题的`_block_other_switchers()`函数，本质上就是GPCT的S2步骤(去冗筛选)——把非目标切换器位置加回walls，等价于"将来自高Epiplexity区的冗余封存入Obscure-Class"。

### 2.3 下一步: 从Oracle到Agent的GPCT迁移

Oracle使用game internals直接知道边界层(哪些是切换器/目标/墙壁)。Agent只有grid信息，需要:

1. **从grid推断BL_c**: 通过差分检测+连通域分析，识别哪些像素块是切换器(会改变属性的)、目标(需要到达的)、墙壁(不可通过的)
2. **推断后的GPCT流程与Oracle完全一致**: type_order枚举 + BFS外区求解
3. **M106(Φ-收敛检测)**: Agent的推断是否收敛——如果grid分析无法确定切换器位置，M106=Fail，触发fallback策略

## 三、灵体几何学 → 路径规划操作语义

### 3.1 五大原型映射

#### Oloid展开 (可展方向 K≡0) → 避免副作用路径

Oloid曲面的高斯曲率处处为零，可以沿直母线方向无拉伸地展开为平面。在ARC中:

- **可展方向** = 不触发非预期属性变化的路径方向
- **非可展方向** = 穿过切换器位置的路径(会触发属性变化)
- **OUA算法** = `_block_other_switchers()`实现: BFS只在"可展区域"(非切换器位置)寻路
- **准可展带** = 允许穿越目标类型切换器(因为那是预期的属性变化)，但阻挡其他类型

**这就是Level 5修复的理论基础**: BFS路径沿Oloid可展方向前进，不撕裂信息(不触发非预期属性变化)。

#### 钢板网 (同胚变换/拓扑守恒) → 状态一致性

钢板网冲压"一片废料都没有"——整体成型，拓扑守恒。

- **拓扑守恒** = 模拟中的状态转换与实际游戏执行一致
- **BFS穿越bug** = 模拟与执行不一致 = 钢板网"有废料" = M88一致性Hash失败
- **修复后** = 模拟准确预测执行 = 钢板网"零废料" = M88通过

#### 三角钻头 (非结合约束干涉) → 移动切换器时间依赖

三角钻头用圆规画方——非线性干涉让矛盾在更高维度抵消。

- **移动切换器** = 位置随步数变化 = 非线性约束干涉
- **时间依赖BFS** = state=(position, step_mod_period) = 在更高维度(加入时间维)消解矛盾
- **Level 4移动旋转切换器** = 4步周期 → 时间维模4 → BFS在5D空间(2D位置+1D时间)求解

#### 正方变三角 (空间折叠/维度重标) → 状态空间重参数化

正方形变三角形——改变维度定义，让起点和终点在高维空间中变得相邻。

- **状态重参数化** = (x, y) → (x, y, step%T, rot, shape, color)
- **维度跃迁** = 把2D网格搜索升级为6D状态空间搜索
- **效果** = 原来不可达的目标(因为不知道何时访问切换器)变为可达(因为状态空间包含了所有信息)

#### 鲁珀特之泪 (奇点攻击/流贯囚禁) → 目标优先级

头部极硬(高约束)、尾部极脆(低约束)。集中攻击头部奇点引发链式崩解。

- **头部奇点** = 最难完成的目标(需要最稀缺的属性组合)
- **链式崩解** = 完成最难目标后，其他目标变得更容易(completed_goals追踪)
- **尾部释放** = 步数补充道具(refill)是"尾部"——低约束但提供能量
- **策略** = 先攻最难目标(头部)，再利用completed_goals释放通路(链式崩解)

### 3.2 OUA算法 → Agent路径规划

```
OUA(M_L2, p_start, p_target):
  Step 1: 计算可展方向(避开非目标切换器)
  Step 2: 沿可展方向积分测地线(BFS寻路)
  Step 3: 构建局部等距(路径映射到动作序列)
  Step 4: 平展计算(在简化空间执行规划)
  Step 5: 验证候选(模拟执行 → M88一致性校验)
  Step 6: 递归(若不一致，重新估计可展方向)
```

这正是Oracle v17的`simulate_route()` + `_block_other_switchers()` + 执行验证的完整流程。

### 4.2 MUS双存对Agent的价值

当Agent无法确定最佳路径时(如移动切换器位置推断不确定)，MUS保持多个候选状态:

```python
# MUS双存: 不确定时保持双路径
if abs(path_a_confidence - path_b_confidence) < threshold:
    # MUS_ACTIVE: 双存不坍缩
    maintain_both_paths(path_a, path_b)
    # 继续传播非冲突超边(NASGA)
    # 等待更多观测信息后再决策
else:
    # 置信度差异大，选择更优路径
    choose_better(path_a, path_b)
```

**应用场景**: Agent从grid推断切换器位置时，可能有多个候选位置。MUS双存保持这些候选，等到Agent实际移动并观测grid变化后，再确认哪个推断正确。

### 4.3 Dead-Zero对Agent搜索的价值

步数是有限资源。当某个搜索分支的步数消耗超过阈值且离目标很远时，Dead-Zero物理熔断:

```python
# Dead-Zero: 步数熔断
if remaining_steps < min_steps_to_goal:
    # 物理熔断，停止该分支搜索
    # valid_seed = 0
    prune_branch()
    # 不写入EpisodeMemory(防"妄念宇宙"增生)
```

**与MUS正交**: Dead-Zero拒绝无据路径(步数不够到目标)，MUS保留有据悖论(两条都能到但不确定哪条更优)。

## 五、IDO信息最优化 → Agent目标函数

### 5.1 ℐ-最优调度

IDO的梯度流沿着信息密度增长最快的子流形前进。在ARC中:

- **ℐ(e)** = 一条路径/动作的信息价值(到目标的进展 × 步数效率)
- **梯度流** = Agent优先探索ℐ增长最快的路径
- **IR不动点** = 到达目标(ℐ达到最大)

### 5.2 ℐ-修正Loss

```
L_total = L_steps + λ · L_ℐ

L_ℐ = -Σ ℐ(edge)  # 鼓励保高ℐ结构(关键决策点)
```

**应用**: Agent的学习目标不仅是"用最少步数完成"，而是"在每个决策点最大化信息获取"——这自然导向GPCT的边界层优先策略。

## 六、认识论倒置 → 理解ARC泛化

### 6.1 认知泰勒展开

文章6提出"物理定律是认知层累涌现的现象"，用泰勒展开描述理论演化:

- T_0(牛顿力学) = 零阶项，在低速宏观完全正确
- T_1(相对论/量子) = 一阶修正
- T_∞(统一理论) = 级数极限

### 6.2 ARC泛化 ≠ OOD

用户之前分享的文章指出"ARC要求的泛化不是普通机器学习的泛化，比OOD要难得多"。认识论倒置给出解释:

- **普通ML的OOD** = 测试集分布偏移，但任务类型相同(都是分类/回归)
- **ARC的泛化** = 每个任务都是一个新的"认知层级"——不是参数偏移，而是结构跃迁
- **泰勒展开类比** = 普通ML在同一阶T_k内做插值；ARC要求从T_k跳到T_{k+1}
- **MUS双存的价值** = 面对新认知层级(新ARC任务)，保持多个假设不急于坍缩 = 保持认知弹性

### 6.3 莫比乌斯环 → Agent自举

Agent从grid推断游戏规则，用推断的规则规划路径，用执行结果验证推断——这是莫比乌斯环:

```
grid观测 → 规则推断 → 路径规划 → 执行 → grid观测 → 规则修正 → ...
```

这个循环不是缺陷，而是自指数学宇宙的定义。Agent通过这个循环逐步"层累"出对游戏规则的理解。
